# Style Guide

Use this guide when the request is about a nostalgic or Winamp-like desktop player.

## Visual direction

- Base the UI on a dark frame with one bright accent.
- Good accents include neon green, electric blue, acid yellow, or hot orange.
- Keep panels boxy and deliberate.
- Use decorative separators, tiny display bars, or geometric trims to make the shell feel designed.

## Typography

- Prefer monospace or tech-leaning display fonts for titles and status text.
- Keep functional labels short: `OPEN`, `PLAY`, `STOP`, `SHARE`, `VOL`.
- Avoid overly soft or editorial typography.

## Layout

- Create a top title zone, a file/status strip, a transport row, a volume row, and a footer/status bar.
- Use decorative space intentionally, not everywhere.
- Let the user locate the current state in under a second.

## Behavior

- Make temporary success states visible but brief.
- Keep playback state obvious in both button labels and status text.
- Treat clipboard or social actions as flavor, not core playback dependencies.

## Constraints

- Do not copy proprietary Winamp skins exactly.
- Do not let decorative elements interfere with controls.
- Keep first versions fixed-size unless responsiveness is part of the request.
